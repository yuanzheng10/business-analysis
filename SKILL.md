# 商业分析顾问 Skill

本 skill 涵盖 35 个经典商业分析框架，分 5 类：
- **战略**：SWOT/TOWS、PEST、五力模型、麦肯锡7S、BCG矩阵、麦肯锡矩阵、平衡计分卡、价值链、3C三角、SPACE矩阵、战略钟、BLM、技术成熟度曲线、S曲线
- **营销**：4P/4C/4R、STP、AIDA/AIDMA、USP、安索夫矩阵、顾客金字塔、品牌健康度、BCG三四规则、生命周期
- **商业模式**：商业模式画布(BMC)、魏朱六要素、业务铁三角
- **组织**：RACI、Greiner成长模型、组织模式画布、人才选育用留
- **思维工具**：麦肯锡七步法、逻辑树/MECE、鱼骨图、PDCA、5W2H、SCP模型

框架详情见 `references/` 目录，按需读取对应文件。

---

## 工作流程

### 第一步：识别问题类型，推荐框架

根据用户描述判断问题类型，从下表选出最合适的 1-3 个框架：

| 问题类型 | 推荐框架 |
|---------|---------|
| 宏观/行业环境 | PEST → 五力模型 |
| 竞争策略 | 五力模型 → SWOT → 3C三角 |
| 内部能力诊断 | 麦肯锡7S → 价值链 |
| 产品/业务组合 | BCG矩阵 → 麦肯锡矩阵 |
| 营销策略制定 | 4P → STP → AIDA |
| 战略选择 | TOWS → 战略钟 → SPACE |
| 商业模式设计 | BMC → 魏朱六要素 |
| 问题拆解 | 麦肯锡七步法 → 逻辑树 → 鱼骨图 |
| 客户/市场分析 | 顾客金字塔 → STP → 马斯洛 |
| 组织诊断 | 麦肯锡7S → RACI → Greiner |
| 技术/产品成熟度 | 技术成熟度曲线 → S曲线 |

如需框架详情，读取 `references/` 中对应文件：
- `references/strategy.md` — 战略类
- `references/marketing.md` — 营销类
- `references/business_model.md` — 商业模式类
- `references/organization.md` — 组织管理类
- `references/thinking_tools.md` — 思维工具类

### 第二步：执行结构化分析

按框架结构逐维度分析，输出格式：

```
## [框架名] 分析

**背景**：[1-2句场景概括]

**分析**：
[按框架维度逐项展开，推断内容标注「推断」]

**结论**：[3-5条关键发现]

**建议**：[3-5条可执行行动]
```

### 第三步：自审纠偏（每次必须执行）

分析完成后，执行以下检查并在报告末尾输出结果：

```
## 自审纠偏

**评分**：[0-100]

**问题清单**：
- [ ] 完整性：各维度是否有实质内容，无占位符？
- [ ] 逻辑一致性：各结论之间是否矛盾？
- [ ] 证据充分性：关键判断有依据还是纯推测？（标注「有据」/「推测」）
- [ ] MECE：问题分解是否穷举且不重叠？
- [ ] 可操作性：建议是否具体可执行，无空泛表述？
- [ ] 偏见：分析是否过于乐观/悲观，或忽略某方视角？
- [ ] 盲点：因信息不足，哪些重要因素未被覆盖？

**修正**：[对有问题的结论给出修正版本]

**待补充信息**：[需要什么额外信息才能让分析更准确]
```

---

## 输出规则

- **推断需透明**：信息不足时标注「基于[假设]推断」
- **结论必须明确**：不能只列问题不给方向
- **自审要真实**：必须找出真正的问题，不能流于形式
- **建议要具体**：每条建议含主语+动词+可衡量目标，禁止"加强管理"这类空话
