# 商业模式框架详情

## <a id="bmc"></a>商业模式画布（BMC）

**定义**：Alexander Osterwalder 提出的九要素商业模式分析工具。

```
九个模块（从右到左填写）：

【右侧：市场端】
① 客户细分（CS）：
   企业服务哪些客户群体？
   - 大众市场 / 利基市场 / 多边市场 / 平台市场

② 价值主张（VP）：中心模块，最重要
   为客户解决什么问题？提供什么价值？
   - 新颖性 / 性能 / 定制化 / 设计 / 价格 / 便捷性

③ 渠道通道（CH）：
   如何触达客户传递价值？
   - 自有渠道 vs 合作渠道
   - 线上 vs 线下

④ 客户关系（CR）：
   与每个客户群建立何种关系？
   - 自助服务 / 专属服务 / 社区 / 共同创造

⑤ 收入来源（RS）：
   从客户身上如何赚钱？
   - 销售收入 / 订阅费 / 使用费 / 许可费 / 广告费

【左侧：运营端】
⑥ 核心资源（KR）：
   提供价值主张所需的关键资产？
   - 实体资源 / 知识资源 / 人力资源 / 财务资源

⑦ 关键业务（KA）：
   运营商业模式必须执行的核心活动？
   - 生产 / 问题解决 / 平台/网络

⑧ 重要合作（KP）：
   哪些业务要外包？哪些资源来自外部？
   - 战略联盟 / 合资 / 供应商关系

【底部：成本】
⑨ 成本结构（CS）：
   运营商业模式的主要成本构成？
   - 成本驱动 vs 价值驱动
   - 固定成本 vs 可变成本

填写顺序建议：
  VP → CS → CH → CR → RS → KR → KA → KP → 成本结构
```

**分析方法**：
- 检查各模块之间的逻辑一致性
- 找出最脆弱的模块（竞争对手最容易复制或颠覆的）
- 评估价值主张与收入来源的匹配度

## <a id="wei_zhu"></a>魏朱六要素商业模式

**定义**：北大魏朱教授提出，从中国商业环境视角分析商业模式的构成逻辑。

```
六个要素：

定位（核心起点）：
  企业满足客户需求的方式
  = 产品/服务 × 目标客户 × 客户需求 × 满足方式

业务系统（运行机制中心）：
  企业选择哪些行为主体作为内/外部利益相关者
  谁做什么，如何协作

关键资源能力：
  支撑交易结构背后的核心资源和能力
  （技术/品牌/数据/关系/专利等）

盈利模式：
  以利益相关者划分的收入结构、成本结构及收支方式
  思考：谁付钱？怎么付？频率如何？

现金流结构：
  企业现金流流入和流出的结构及形态
  是否有正向现金流循环？

企业价值（最终输出）：
  未来净现金流的贴现
  = 规模 × 效率 × 持续性

与BMC的区别：
  魏朱模型更强调"定位→运营机制→企业价值"的因果逻辑链
  BMC更像静态清单，魏朱更像动态系统图
```

## <a id="iron_triangle"></a>业务铁三角

**定义**：从产品、核心业务职能、市场三个维度思考公司从事什么业务。

```
三个顶点：

产品（Product）：
  公司为客户解决什么问题？
  产品的核心价值是什么？

核心业务职能（Function）：
  公司在价值链中扮演什么角色？
  - 研发型 / 制造型 / 渠道型 / 服务型 / 平台型

市场（Market）：
  我们的目标市场在哪里？
  - 地理范围 / 行业 / 客群规模

使用方法：
  1. 明确三个顶点各自的答案
  2. 检查三者之间的一致性（是否相互支撑？）
  3. 识别不一致处 = 业务定位需要调整的地方

示例（对比两种SaaS公司）：
  公司A：产品=项目管理工具 / 职能=研发型 / 市场=中国中小企业
  公司B：产品=项目管理工具 / 职能=渠道型 / 市场=大型制造企业
  → 同样的产品，不同的职能和市场定位，是完全不同的商业模式
```
