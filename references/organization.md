# 组织管理框架详情

## <a id="raci"></a>RACI 模型

**定义**：明确组织变革或项目中各角色责任的工具。

```
四个角色：

R（Responsible 谁负责执行）：
  具体执行任务的角色
  负责操控项目、解决问题
  一个任务可以有多个R

A（Accountable 谁批准/对结果负全责）：
  对任务最终结果负责的人
  只有经他/她同意，项目才能推进
  每个任务只能有一个A

C（Consulted 咨询谁）：
  拥有完成任务所需信息或专业能力的人
  双向沟通，在执行前需要咨询其意见

I（Informed 告知谁）：
  需要及时被通知结果的人
  单向沟通，不需要征求意见

RASCI（扩展版）增加了：
  S（Supportive 谁提供支持）：
  提供信息资源、辅助执行任务的人员

使用方法：
  1. 列出所有关键任务/决策（行）
  2. 列出所有相关角色/部门（列）
  3. 在交叉格填入 R/A/C/I
  4. 检查：每行是否有且仅有一个A？
  5. 识别空白格（无人负责）和冲突格（多人都是A）
```

**常见问题诊断**：
- 某列全是A → 此人权力过于集中，成为瓶颈
- 某行没有A → 这个任务没有最终负责人，容易失控
- 太多C → 决策流程过慢，需要精简咨询范围

## <a id="greiner"></a>Greiner 组织成长模型

**定义**：企业在不同发展阶段会经历不同的危机和挑战。

```
五个发展阶段：

阶段1 创业期（在创业中发展）：
  特征：创始人主导，非正式管理，快速试错
  危机：领导力危机（需要专业管理者接手）

阶段2 引导期（在指导下发展）：
  特征：引入职业经理人，建立规范管理
  危机：自主危机（一线员工感到缺乏授权）

阶段3 授权期（在授权下发展）：
  特征：分权管理，各部门/事业部独立运营
  危机：控制危机（总部失去对各部门的把控）

阶段4 协调期（在协调下发展）：
  特征：重建协调机制，设立职能部门管控
  危机：官僚主义危机（流程繁琐，效率下降）

阶段5 合作期（在合作下发展）：
  特征：矩阵式/团队式组织，强调协作创新
  危机：未知（心理饱和？组织过于复杂？）

使用方法：
  判断企业当前处于哪个阶段
  → 预判即将到来的危机类型
  → 提前布局应对措施
  → 避免"用上一阶段的成功方法解决当前阶段的问题"
```

## <a id="org_canvas"></a>组织模式画布

**定义**：系统性设计企业组织架构的工具，六大关键维度。

```
六个维度：

挑战&问题（上层）：
  在业务远景下，有哪些核心挑战需要突破？
  当前组织面临的主要障碍是什么？

产品与服务（左下）：
  我们给客户提供什么样的产品和服务？
  这决定了需要哪种类型的组织能力

未来愿景（中上）：
  组织的长期目标/使命是什么？
  愿景决定了组织设计的方向

企业名称/定位（中心）：
  我们是谁？一句话简单介绍
  定位决定了组织的基本形态

策略和机会（右上）：
  我们的竞争策略是什么？
  如何实施这些策略？
  策略决定了优先投入哪些组织能力

市场定位（右下）：
  我们目前在市场的位置如何？
  竞争格局如何影响组织设计？

使用方法：
  先填中心（定位），再向外延伸
  检查六个维度之间的逻辑一致性
  找出"组织设计与战略不匹配"的地方
```

## <a id="talent_model"></a>人才选育用留

**定义**：企业人才管理的完整闭环流程。

```
六个环节：

选（人力资源规划 + 招聘配置）：
  HR规划：预测业务发展需要什么样的人才？多少人？
  招聘：如何找到合适的人？
  配置：如何将合适的人放在合适的位置？

育（职业生涯发展 + 培训管理）：
  职业通道：为员工设计清晰的晋升路径
  培训体系：岗前培训/在职培训/管理力培养
  关键：让员工看到成长空间

用（绩效管理）：
  目标设定：OKR/KPI的制定
  过程管理：辅导与反馈
  绩效评估：公平、透明、有依据
  关键：让优秀的人得到充分发挥

留（薪酬与激励）：
  薪酬结构：固定薪资/绩效奖金/长期激励
  非物质激励：文化/认可/成就感
  关键：留住想留的人，淘汰该淘汰的人

核心诊断问题：
  选：我们招的人是否符合业务发展需要？
  育：员工是否在持续成长？
  用：人才是否用在了最能发挥价值的地方？
  留：关键人才流失率如何？原因是什么？
```
