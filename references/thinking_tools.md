# 思维工具框架详情

## <a id="mckinsey_7steps"></a>麦肯锡七步成诗法

**定义**：麦肯锡公司提出的解决问题的基本逻辑框架。

```
第一步 陈述问题：
  • 明确要解决的基本问题
  • 具体、有内容的描述（避免模糊）
  • 清楚列示问题涉及的各方面信息

第二步 分解问题：
  • 遵循 MECE 原则（完全穷举，相互独立）
  • 采用树状图/逻辑树分解
  • 不断自出假设并修正

第三步 确定关键问题：
  • 用 80/20 法则发现关键驱动因素
  • 持续头脑风暴
  • 聚焦最重要的2-3个子问题

第四步 制定工作计划：
  • 明确效率、成品、责任
  • 细化到人、时间、具体内容

第五步 进行关键分析：
  • 以假设为前提，事实为依据，结构化论证
  • 常用方法：因果分析、比例分析、标杆比较、趋势分析、模型分析

第六步 形成结构性结论：
  • 总结分析结果
  • 按结构化方式组织论点
  • 提出解决方案的建议

第七步 形成说服力方案：
  • 结构化表达（金字塔原理）
  • 使用图形（表）清晰生动表达
  • 尽量简洁，突出重要性内容

注意：假设不成立时，要从头再来（循环反复）
```

## <a id="logic_tree"></a>逻辑树 / MECE 原则

**定义**：将问题的所有子问题分层罗列，保证分析完整性。

```
MECE 原则：
  M = Mutually Exclusive（相互独立）：分类之间没有重叠
  CE = Collectively Exhaustive（完全穷举）：所有分类加起来覆盖全部

逻辑树类型：
  问题树：识别问题的所有组成部分
  原因树：追溯问题的所有根本原因
  方案树：穷举所有可能的解决方案

使用逻辑树的四大好处：
  1. 分解问题，使工作可管理化
  2. 在不同部分之间设置优先级
  3. 把职责分解到个人
  4. 确保解决问题的完整性

常见的MECE分解框架：
  按流程：输入→处理→输出
  按时间：过去→现在→未来
  按空间：内部→外部
  按要素：人→财→物→信息
  按问题类型：什么→为什么→如何
```

## <a id="fishbone"></a>鱼骨图（因果分析图）

**定义**：通过对原因的详细罗列找出影响结果的方法，避免遗漏。

```
使用步骤：
  1. 在鱼头写下要分析的问题/结果
  2. 确定主要原因类别（大骨，通常4-6个）
     制造业常用：人/机/料/法/环（5M）
     服务业常用：人员/流程/政策/设备/外部
  3. 在每个类别下细化具体原因（小骨）
  4. 继续追问"为什么"深挖原因
  5. 圈出最关键的根本原因

示例（销售额下降的鱼骨图）：
  主骨：销售额下降
  产品大骨：→功能不满足需求 →质量问题 →价格过高
  营销大骨：→目标客户不准 →触达渠道错误 →内容缺乏吸引力
  销售大骨：→团队能力不足 →激励机制问题 →跟进不及时
  外部大骨：→市场萎缩 →竞争加剧 →政策变化
```

## <a id="pdca"></a>PDCA 循环

**定义**：闭环问题解决方法，特别适用于质量管理和运营改进。

```
四个阶段：
  P（Plan 计划）：
    1. 找问题
    2. 找原因（鱼骨图/逻辑树）
    3. 找要因（确定关键原因）
    4. 订计划（目标/措施/时间/责任人）

  D（Do 执行）：
    5. 执行计划中的措施

  C（Check 检查）：
    6. 检查监控执行效果，与目标对比

  A（Action 行动/纠正）：
    7. 纠正问题（解决偏差）
    8. 提出新问题（进入下一轮循环）

PDCA 特点：
  • 永不停止（每次循环解决一个问题，再开始下一个）
  • 螺旋上升（每轮循环后水平都有提升）
  • 适用于任何管理场景：营销/服务/质量/研发
```

## <a id="5w2h"></a>5W2H 分析

**定义**：通用问题分析框架，适用于任何工作计划或问题分析。

```
七个维度：
  Why（为什么）：明确目标，为何要做这件事？
  What（做什么）：具体任务内容是什么？
  Who（谁来做）：责任人是谁？
  When（什么时间）：截止时间/时间节点？
  Where（在哪里）：执行地点/场景？
  How（如何做）：具体方法/步骤？
  How Much（做多少）：数量/预算/规模？

示例（制定营销活动方案）：
  Why: 提升新产品认知度，获取1000个注册用户
  What: 线上内容营销+限时免费体验活动
  Who: 市场部3人主导，销售部配合
  When: 下月1日-30日
  Where: 微信/抖音/行业社群
  How: 发布10篇深度文章+5个短视频+1场直播
  How Much: 预算5万元，目标CPL≤50元
```

## <a id="scp"></a>SCP 分析模型

**定义**：当企业遭受外部冲击时，从行业结构→企业行为→经营绩效三个角度分析影响。

```
分析链条：
  外部冲击(Shock) → 行业结构(Structure) → 企业行为(Conduct) → 经营绩效(Performance)

示例（AI冲击对咨询行业的影响）：
  冲击：大模型AI快速普及
  
  行业结构变化：
    • 进入壁垒降低（AI工具使个人可提供咨询服务）
    • 替代品增加（AI可直接回答分析问题）
    • 竞争格局从寡头向多极化演变
  
  企业行为变化：
    • 大型咨询公司加速AI工具开发
    • 中小公司选择细分行业深耕
    • 定价策略调整（日费下降，项目制增加）
  
  经营绩效影响：
    • 标准化分析类业务利润率下降
    • 战略决策类高端业务仍有溢价空间
    • 人均产值可能提升（AI提效）

应用场景：
  政策变化/技术冲击/疫情/汇率波动对企业的传导影响分析
```
